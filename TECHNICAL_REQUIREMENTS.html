2. Softwarové požadavky pro produkční nasazení

2.1 Server (Backend/Frontend - Web)

Operační systém

  • Linux (Ubuntu 20.04 LTS / 22.04 LTS / 24.04 LTS) - DOPORUČENO pro produkci
  • macOS (12.0+) - vhodný pro development a testování
  • Windows Server 2019+ - s WSL2 pro Linux kompatibilitu

Runtime Environment

  • Node.js 18.17.0+ (ideálně 20.x LTS nebo 22.x)
  • npm 9+ nebo pnpm 8+
  • Package manager MUSÍ být konzistentní v celém deployment pipeline

Databázový server

  • PostgreSQL 13+ (Supabase cloud hosted - doporučeno)
  • Minimálně 100 MB úložného prostoru na začátek
  • Max. connections: 50-100 pro aplikaci (dle konfigurace)
  • Automated backups: Minimálně denně
  • Replication: Point-in-time recovery (PITR)

Webový server

  • Next.js vestavěný server (doporučeno s Vercel hosting)
  • NEBO: Nginx/Apache reverse proxy před Next.js instancí
  • SSL/TLS certifikát (Let's Encrypt, auto-renew)
  • HTTP/2 a gzip komprese POVINNÉ

Dodatečné služby

  • PM2 - Process manager pro Node.js (load balancing, auto-restart)
  • Redis (optionální) - Pro caching a session storage (Upstash)
  • Monitoring agent - New Relic, DataDog, nebo PM2 Plus

3. Hardwarové požadavky

3.1 Minimální konfigurace (pro 10-20 operačních sálů)

Server (Linux/Ubuntu)

  • CPU: 2 jádra (x86_64), 2 GHz
  • RAM: 2 GB (doporučuje se 4 GB)
  • Disk: 20 GB SSD (10 GB pro OS, 10 GB pro aplikaci)
  • Síť: 10 Mbps (minimálně 100 Mbps doporučeno)
  • Vhodné pro: Malá klinika, testing, development

3.2 Doporučená konfigurace (pro 20-50 sálů, 50-100 concurrent uživatelů)

Server (Linux/Ubuntu)

  • CPU: 4 jádra, 2.5+ GHz (Intel Xeon / AMD EPYC)
  • RAM: 8 GB (doporučuje se 16 GB pro cache a session storage)
  • Disk: 50-100 GB SSD (NVMe doporučeno)
  • Síť: 100 Mbps (ideálně 1 Gbps)
  • Vhodné pro: Střední nemocnice, standard produkční nasazení

3.3 Enterprise konfigurace (pro 50+ sálů, 200+ concurrent uživatelů)

Primární server

  • CPU: 8+ jader, 2.5+ GHz
  • RAM: 32 GB
  • Disk: 200+ GB SSD NVMe (RAID 1 minimum)
  • Síť: 1 Gbps

Backup/Replica server

  • Stejná konfigurace jako primární
  • PostgreSQL replication pro failover
  • Load balancer (Nginx/HAProxy)
  • Vhodné pro: Velké nemocnice, kritické aplikace
Důležitá poznámka: Všechny hardwarové konfigurace by měly být s headroom minimálně 20% pro peak load a budoucí growth. SSD disk je POVINNÝ (HDD není doporučeno).

9. Bezpečnostní požadavky

9.1 TLS/SSL šifrování

  • Minimálně: TLS 1.2 (ale TLS 1.2 je zastaralý)
  • Doporučeno: TLS 1.3 (nejnovější, nejbezpečnější)
  • Certifikát: Let's Encrypt (zdarma, auto-renewal)
  • HSTS: Povinné (Strict-Transport-Security header)
  • Certificate pinning: Pro mobilní aplikace (doporučeno)

9.2 Hesla

Parametr Požadavek
Minimální délka 12 znaků
Complexity Velká písmena + čísla + speciální znaky (!@#$%)
Hashing algoritmus bcrypt (s salt) - POVINNÉ
Password expiration 90 dní (pro adminy), nepovinné pro ostatní
Password history Neumožnit posledních 5 hesel

9.3 Session management

Parametr Hodnota
Session timeout 30 minut neaktivity (admin), 2 hodiny (běžný uživatel)
Session storage Server-side (Supabase), NIKDY na client
HTTPS only Ano (povinné)
Secure cookies Ano (flag Secure, HttpOnly, SameSite)
Session ID length Minimálně 128 bits (16 bytes)

9.4 Rate limiting a DDoS ochrana

  • Login attempts: 5 pokusů za 15 minut (pak lockout)
  • API endpoints: 100 requests za minutu na uživatele
  • WebSocket: 50 messages za sekundu
  • DDoS mitigation: CloudFlare nebo Akamai

9.5 Datová bezpečnost

  • Data at rest: Šifrování na úrovni databáze (Supabase poskytuje)
  • Data in transit: TLS 1.3
  • Backup encryption: AES-256
  • Access control: Role-Based Access Control (RBAC)
  • Audit logging: Všechny citlivé operace se logují

9.6 Compliance a regulace

  • GDPR: Splnění (zvláště relevantní v EU)
  • HIPAA: Splnění (zdravotnické údaje)
  • ISO 27001: Doporučeno (informační bezpečnost)
  • Regular security audit: Minimálně roční
  • Penetration testing: Ročně nebo po maior updatu

10. Produkční nasazení - kroky

10.1 Server setup

  1. Vytvořit Linux VM (Ubuntu 22.04 LTS) u cloud providera (AWS, Linode, DigitalOcean)
  2. SSH klíče, security groups, firewall konfigurace
  3. Nainstalovat Node.js 20 LTS: 
    curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
  4. Nainstalovat Nginx (reverse proxy): 
    sudo apt-get install -y nginx
  5. Nainstalovat PM2 (process manager): 
    sudo npm install -g pm2
  6. Setup SSL/TLS certifikát (Let's Encrypt): 
    sudo apt-get install -y certbot python3-certbot-nginx
sudo certbot certonly --nginx -d aplikace.cz
  7. Vytvořit .env.production se Supabase klíči
  8. Přidat monitoring (PM2 Plus, Sentry, New Relic)

10.2 Aplikace build a deployment

cd /opt/app
git clone https://github.com/medrox/operatingroommanagement.git .
npm ci --production
npm run build
pm2 start "npm start" --name "app" --instances max
pm2 start "npm start" -- --port 3001 (pro scaled instanci)
pm2 save
pm2 startup
pm2 logs

10.3 Nginx konfigurace (reverse proxy)

upstream next_app {
    server localhost:3000;
    server localhost:3001;
}

server {
    listen 80;
    server_name aplikace.cz;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name aplikace.cz;
    
    ssl_certificate /etc/letsencrypt/live/aplikace.cz/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/aplikace.cz/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    
    location / {
        proxy_pass http://next_app;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 30s;
    }
}

# Počkat na úspěšný start
