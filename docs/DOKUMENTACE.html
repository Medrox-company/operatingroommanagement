Prehled aplikace

1.1 O aplikaci

Operating Room Management System je komplexni webová aplikace pro efektivni rizeni operačních sálů ve zdravotnických zařízeních. Aplikace umožňuje reálné sledování stavu jednotlivych sálů, správu pracovních času, automaticke notifikace operacniho personálu a detailni analýzu efektivity provozu.

✓

Všechny statistiky z realnych dat

Aplikace nepoužívá žádná mockovaná data. Všechny metriky vychází z reálné databáze a historických záznamů.

1.2 Hlavni funkcionality

📋
Real-time monitoring
Live dashboard s aktualni stavem vsech sálů, animovane přechody stavu a okamzita aktualizace.
⏱
Timeline modul
Casova osa s vizualizaci operaci, fazi a planovanych zakroku pro cely den.
🔔
Notifikacni system
Automatické upozorneni pri pozdnich prichodech, nepripravenych pacientech a dalsich problemech.
📊
Pokrocile statistiky
Detailní analyza vyuziti sálů, efektivity, casovych rad a predikci.
👥
Sprava personálu
Evidence lekaru, sester, anesteziologu s jejich dostupnosti a smenami.
⚙
Systemova nastaveni
Flexibilni konfigurace workflow, pracovních hodin, oddelen a kontaktu.
📅
Planovani operaci
Kalendar s planovanymi zakroky, prioritami a automatickym rozvrhem.
✅
Bezpečnostní checklisty
WHO Surgical Safety Checklist (Sign-In, Time-Out, Sign-Out) integrovaný do workflow.

3.2 Klicova tabulka: operating_rooms

CREATE TABLE operating_rooms (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  department TEXT,
  status TEXT DEFAULT 'idle',
  current_step_index INTEGER DEFAULT 0,
  phase_started_at TIMESTAMPTZ,
  operation_started_at TIMESTAMPTZ,
  estimated_end_time TIMESTAMPTZ,
  
  -- Pocty operaci
  operations_24h INTEGER DEFAULT 0,
  queue_count INTEGER DEFAULT 0,
  completed_operations JSONB,
  
  -- Specialni priznaky
  is_emergency BOOLEAN DEFAULT FALSE,
  is_locked BOOLEAN DEFAULT FALSE,
  is_paused BOOLEAN DEFAULT FALSE,
  is_septic BOOLEAN DEFAULT FALSE,
  
  -- Prirazeny personal
  doctor_id TEXT,
  nurse_id TEXT,
  anesthesiologist_id TEXT,
  current_patient_id TEXT,
  current_procedure_id TEXT,
  
  -- Financni data
  hourly_operating_cost NUMERIC,
  
  -- Rozvrh
  weekly_schedule JSONB,
  status_history JSONB,
  
  -- Metadata
  sort_order INTEGER,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

3.3 Tabulka historie: room_status_history

ⓘ

Dulezite pro statistiky

Tato tabulka je zakladem vsech statistickych vypoctu. Kazda zmena stavu sálů se zaznamenava.

CREATE TABLE room_status_history (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  operating_room_id TEXT NOT NULL REFERENCES operating_rooms(id),
  event_type TEXT NOT NULL,     -- 'operation_start', 'step_change', 'operation_end'
  step_name TEXT,               -- Nazev workflow faze
  step_index INTEGER,           -- Index faze (0, 1, 2, ...)
  duration_seconds INTEGER,     -- Delka trvani v sekundach
  timestamp TIMESTAMPTZ NOT NULL,
  metadata JSONB,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Indexy pro rychle dotazy
CREATE INDEX idx_room_status_history_room ON room_status_history(operating_room_id);
CREATE INDEX idx_room_status_history_time ON room_status_history(timestamp);
CREATE INDEX idx_room_status_history_type ON room_status_history(event_type);

Software jako zdravotnicsky prostredek (SaMD) urceny pro podporu klinickych rozhodnuti v operacnim prostredi.

Pravidlo Aplikace Vysledek

5.3 Regulační pozadavky

  • CE znaceni - Povinne pro uvedeni na trh EU
  • QMS (ISO 13485) - System managementu kvality
  • Technická dokumentace - Dle Annex II a III MDR
  • Klinicke hodnoceni - Dle Annex XIV MDR
  • Post-market surveillance - Kontinualni sledování
  • UDI - Unikatni identifikator zarizeni

5.4 Intended Purpose

Ucel pouziti: Operating Room Management System je software urceny pro podporu koordinace a monitorovani operačních sálů ve zdravotnických zařízeních. System umožňuje:

  • Real-time sledování stavu operačních sálů
  • Koordinaci zdravotnickeho personálu
  • Spravu workflow operačních zakroku
  • Generovani statistik a analytickych reportu
  • Notifikaci pri kritickych stavech

Cilova skupina: Zdravotnicky personal (chirurgove, anesteziologove, sestry, koordinatori)

Prostredi pouziti: Nemocnice, chirurgicka centra, ambulantni operacni saly

Custom auth s bcrypt hašováním, session tokens, automaticke odhlaseni.

Row Level Security (RLS) na vsech tabulkach, role-based access.
Sifrovani
HTTPS/TLS 1.3, sifrovani dat v klidu (AES-256).
