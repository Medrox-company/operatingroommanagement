1. PŘEHLED SYSTÉMU

1.1 Charakteristika Aplikace

Název: Operating Room Management System (ORMS) v3.0

Typ: Webová a mobilní aplikace pro management operačních sálů

Kategorie: Enterprise healthcare management system

Architektura: Full-stack Web (Next.js 16) + Mobilní (iOS/Android)

Databáze: Supabase (PostgreSQL) s realtime synchronizací

Bezpečnost: End-to-end HTTPS/TLS 1.3 + Role-Based Access Control

1.2 Hlavní Funkcionality

  • Dashboard s přehledem všech operačních sálů v reálném čase
  • Správa plánů operací s automatickým rozvrhem
  • Evidence personálu a jejich dostupnosti
  • Monitoring stavu a spotřeby materiálu v sálech
  • Notifikace a alerting pro operační tým
  • Integrovaný chat pro komunikaci mezi zaměstnanci
  • Reporting a analýzy účinnosti sálů
  • Offline mód pro zachování funkčnosti

4. SÍŤOVÉ POŽADAVKY A KONEKTIVITA

4.1 Šířka Pásma (Bandwidth)

Počet Sálů Počet Uživatelů Minimálně Doporučeno Ideálně
5-10 5-10 5 Mbps 20 Mbps 50 Mbps
10-20 15-30 10 Mbps 50 Mbps 100 Mbps
20-50 30-100 25 Mbps 100 Mbps 500 Mbps
50+ 100+ 100 Mbps 500 Mbps 1 Gbps

4.2 Latence (Ping / Odezva)

Minimálně (Aplikace funguje): < 200 ms

Doporučeno (Normální provoz): < 100 ms

Ideálně (Plynulá realtime): < 50 ms

Pro kritické realtime (Realtime Subscriptions): < 150 ms

Poznámka: Když latence překročí 300 ms, realtime aktualizace status sálů se mohou zpoždět. To může vést k nekonzistenci dat. Doporučuje se monitorovat latenci během provozu.

4.3 Stabilita Síťového Spojení

Parametr Minimální Doporučeno Ideální
Uptime (Dostupnost) 95% (3.6 h výpadku/měsíc) 99% (7 min výpadku/měsíc) 99.9% (43 sec/měsíc)
Packet Loss (Ztráta dat) < 2% < 0.5% < 0.1%
Jitter (Variabilita latence) < 100 ms < 50 ms < 20 ms

4.4 Typ Připojení

Kabelové připojení (Wired): Ethernet (RJ45) Cat5e nebo novější. Zajišťuje nejstabilnější a nejrychlejší spojení pro desktop počítače.
WiFi: WiFi 5 (802.11ac) nebo WiFi 6 (802.11ax). Minimálně 2.4 GHz pásmová frekvence, ideálně 5 GHz. Pro mobilní zařízení.
5G Mobilní: Ideální pro tablety a smartphony mimo kancelář. Latence ~ 20-50 ms, bandwidth 100+ Mbps.
VPN (Virtuální Privátní Síť): Při připojení z vně sítě nemocnice. Doporučuje se IPSec nebo OpenVPN.

6. BEZPEČNOST A COMPLIANCE POŽADAVKY

6.1 Šifrování Dat

Data v Přenosu (In Transit):

  • Protokol: HTTPS TLS 1.3 (povinně) nebo TLS 1.2 (minimum)
  • Šifra: AES-256-GCM
  • Certifikát: Platný SSL/TLS certifikát vydaný trusted CA
  • HSTS (HTTP Strict Transport Security): Povinně aktivní

Data v Klidu (At Rest):

  • Databáze: PostgreSQL s šifrováním na úrovni databáze (Supabase)
  • Backupy: Šifrované pomocí AES-256
  • Hesla: Hashována PBKDF2 nebo bcrypt algoritmy

6.2 Autentifikace a Autorizace

Bezpečnostní Prvek Implementace
Přihlášení Email + heslo (povinné). Podpora 2FA (Two-Factor Authentication) doporučena.
Session Timeout 30 minut neaktivity pro administrátory, 2 hodiny pro běžné uživatele
Role-Based Access Control (RBAC) Admin, Manager, Zdravotník, Personál - různé úrovně přístupu
Session Storage Server-side (Supabase), ne localStorage
Cookie Security HttpOnly, Secure, SameSite=Strict

6.3 Rate Limiting a DDoS Ochrana

Přihlašovací Pokusy: Maximálně 5 pokusů za 15 minut. Poté zablokován účet na 30 minut.

API Rate Limit: 100 requests za minutu na uživatele, 1000 requests za minutu na IP adresu.

WebSocket Connections: Maximálně 50 messages za sekundu na jednoho uživatele.

DDoS Ochrana: Cloudflare nebo podobná služba doporučena pro produkci.

6.4 GDPR a Regulace

Důležité Poznámky:

  • GDPR (Evropská Unie): Všechna pacientská data jsou osobní údaje a musí být chráněna dle GDPR. Vyžaduje se právní základ pro zpracování.
  • HIPAA (USA): Pokud se aplikace používá v USA, musí splňovat HIPAA compliance.
  • Zdravotnická Bezpečnost (Česko): Musí splňovat nařízení České republiky o ochraně zdravotnických dat.
  • Audit Logging: Všechny přístupy k pacientským datům musí být logovány a archivovány minimálně 7 let.

8. TECHNICKÉ SPECIFIKACE APLIKACE

8.1 Backend Architektura

Framework: Next.js 16 (API Routes + Server Components)

Runtime: Node.js 20 LTS+

Databáze: PostgreSQL 13+ (via Supabase)

Realtime: Supabase Realtime (WebSocket-based)

Authentication: JWT tokens + Session-based

Cache: Redis (optional, pro high-traffic scénáře)

8.2 Frontend Architektura

Framework: React 18+ s TypeScript

State Management: React Context API + Custom Hooks

Styling: Tailwind CSS v3+

UI Components: Custom React components + shadcn/ui

Animace: Framer Motion

HTTP Client: Fetch API + SWR pro caching

8.3 Mobilní Aplikace

Typ: Webová aplikace optimalizovaná pro mobilní zařízení (PWA)

Framework: React Native / Expo (volitelně)

iOS Support: iOS 14.0+

Android Support: Android 10+ (API 29+)

Offline Mode: Service Workers + LocalDB

Push Notifications: Firebase Cloud Messaging (FCM)
